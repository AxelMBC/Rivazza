# Design — accurate track limits from the AI spline

## Context

The map shows driven lines floating in empty space. `map.png` was investigated and
rejected: it is a constant-width stroke around the AI line (measured across six stock
tracks: width ±0.4–0.5 m, correlation with real track width ≈ 0). The real source of
track limits shipped with every track is `ai/fast_lane.ai` — AC's AI spline, where each
point carries `sideLeft`/`sideRight`, the measured distances to the left/right track
edge. This is the data the game's own AI uses to stay on track.

Local validation (all 42 installed layouts, stock + mods):

- Every playable track parses with plausible widths (median 6–20 m, no bad records).
- The only failures are correct rejections: uninstalled DLC stubs (no file), `drift`'s
  12-byte placeholder, and `shuto_revival_project_beta` shipping a `fast_lane.ai`
  copied verbatim from Silverstone (caught by bounds cross-check).
- Handedness was resolved empirically: at corner apexes a racing line hugs the inside
  edge, so the smaller side distance must face the bend. Voting across all circuits'
  sharpest corners gives 82.8% for **driver-left = `(dz, -dx)`** in world XZ (100% of
  circuits agree by majority; the only sub-50% "tracks" are straight drag strips with
  no apexes). A rendered overlay of the computed ribbon on Magione confirms a clean,
  non-crossing closed loop.

## Goals / Non-Goals

**Goals:**

- Draw the actual track surface (asphalt ribbon + edge lines) under the driving lines,
  accurate at corner level, for every track that ships a usable `fast_lane.ai`.
- Degrade gracefully: any parse/sanity failure yields exactly today's behavior.
- Keep the render loop's dirty-gating and layer caching intact.

**Non-Goals:**

- Pit lane rendering — `pit_lane.ai` carries all-zero side widths, so there is no pit
  ribbon to draw (verified on magione/imola).
- kn5 road-mesh extraction. More exact in theory, but it describes *paved area*
  (including runoff), not sporting track limits, and costs a full model parser.
- Cut detection / track-limits penalties from telemetry.
- Any change to `map.png` handling (stays never-drawn).

## Decisions

### 1. Source: `fast_lane.ai` side distances

Chosen over kn5 mesh parsing (heavy; semantically paved-area, not track limits) and
`map.png` (disproven). The AI-line sides are generated by Kunos's track tooling from
the road mesh and define the drivable corridor — the right semantics for "track
limits", available for effectively every track, parseable in ~50 lines.

### 2. Binary format and parsing rules

Little-endian: `int32 version(==7)`, `int32 count`, 2 × `int32` (lap time / sample
count, unused), then `count` × 20-byte points `{float x, y, z, length; int32 id}`,
then `int32 extraCount` (must equal `count`), then `count` × 72-byte extra records —
18 floats of which index 5 is `sideLeft` and index 6 is `sideRight`. A grid lookup
section follows; it is ignored (files are larger than header+points+extras). All
structural checks are hard gates: wrong version, short file, or `extraCount ≠ count`
⇒ no edges. Same garbage-tolerant philosophy as `parsers.ts`/`carAssets.ts`: never
throw on bad content, return null and log.

### 3. Edge computation

For point `i` with unit travel direction `d = normalize(p[i+1] − p[i])` (wrapping for
closed loops): `leftEdge = p + (d.z, −d.x) · sideLeft`, `rightEdge = p − (d.z, −d.x) ·
sideRight`. Sides are clamped to [0, 50] m and passed through a 3-wide median filter
to kill isolated spikes without flattening genuinely wide sections (drag strips,
Nordschleife tourist merge). Degenerate consecutive duplicates keep the previous
direction.

### 4. Sanity gates before serving

- ≥ 50 points and ≥ 70% of points with a positive total width, else reject (catches
  stubs and side-less files).
- When `map.ini` exists: ≥ 80% of spline points must project inside the map's world
  rect (with 10% margin), else reject — catches mod tracks shipping another track's
  AI file (shuto case). Without `map.ini` there is nothing to check against; serve.
- Closed-loop flag: `distance(first, last) < 30 m`. Open splines (hillclimbs like
  trento-bondone) don't get a closing segment.

### 5. Wire format

`GET /api/track-map/edges` → `{ closed: boolean, left: [x, z][], right: [x, z][] }`,
coordinates rounded to centimeters (halves the JSON size; sub-pixel at any zoom).
404 when unavailable — mirroring meta/image endpoint behavior. Fetched once per
session alongside meta. Worst local case (Nordschleife endurance, 16.4k points) is a
~600 KB one-time localhost fetch. `SessionInfo` gains `edgesAvailable` (additive,
both `types.ts` mirrored).

### 6. Rendering

A third offscreen layer (`trackLayer`) below `lapsLayer`/`currentLayer`, keyed by
`projKey` only (edges are static per session): filled ribbon polygon (left polyline
forward, right polyline reversed) in a surface tone slightly lighter than the panel
background, plus 1 px edge strokes in muted white — the driving lines and pedal
colors must keep dominating visually. Canvas constants, matching the file's existing
color-constant convention. Ribbon renders in both map.ini mode and the new
edges-only mode; hover picking, legend, and dot draw unchanged on top.

### 7. Viewport when edges exist without `map.ini`

Edge bounds (plus the existing `VIEW_MARGIN`) fix the view from the first frame —
same "camera never moves" guarantee as map.ini mode, replacing the anchored heuristic
for this case. The heuristic camera remains only when neither asset exists.

## Risks / Trade-offs

- [Handedness wrong on some exotic content] → validated across every installed layout;
  a mistake shows as ribbon offset at asymmetric corners, and the fix is a single sign
  flip in one function.
- [Mod tracks with junk AI files render nonsense] → structural + width + bounds gates
  reject them; rejection is always silent fallback to today's behavior.
- [Big AI files (30 MB) parsed at session start] → single buffered read + typed reads,
  tens of ms, once per session change; result cached per track+config while the
  session lasts.
- [Ribbon polygon self-intersects where side data spikes] → median filter + clamps;
  worst remaining case is a local visual artifact in the fill, never a crash.
- [Payload size on huge tracks] → cm rounding; one-time fetch on localhost.

## Open Questions

None blocking. Pit-lane visualization dropped (no side data — see Non-Goals).
